## 📝 커밋 메시지 형식

[Conventional Commits](https://www.conventionalcommits.org/) 명세를 따릅니다:

```
<타입>[선택적 스코프]: <설명>

[선택적 본문]

[선택적 푸터]
```

### 구조 규칙
- **헤더**: `<타입>[선택적 스코프]: <설명>`
  - 최대 50자
  - 타입과 스코프는 소문자
  - 설명은 명령형
  - 끝에 마침표 없음

- **본문** (선택적):
  - 72자에서 줄바꿈
  - 무엇을, 왜 했는지 설명 (어떻게가 아닌)
  - 헤더와 빈 줄로 구분

- **푸터** (선택적):
  - 이슈 참조, 중대한 변경사항
  - 본문과 빈 줄로 구분

## 🏷️ 커밋 타입

### 주요 타입 (필수)

#### `feat` - 새로운 기능
- 사용자를 위한 새로운 기능
- 새로운 API 엔드포인트
- 새로운 UI 컴포넌트
- 새로운 AI 모델 기능

```bash
feat(auth): JWT 토큰 갱신 메커니즘 추가
feat(ui): 다크 모드 토글 구현
feat(ai): 감정 분석 모델 추가
```

#### `fix` - 버그 수정
- 기존 기능의 수정
- 보안 패치
- 성능 수정
- UI/UX 개선

```bash
fix(api): 사용자 서비스의 메모리 누수 해결
fix(ui): 모바일에서 버튼 정렬 수정
fix(ai): 모델 예측의 엣지 케이스 처리
```

#### `docs` - 문서
- 문서 업데이트
- README 변경
- API 문서
- 코드 주석

```bash
docs(api): 인증 엔드포인트 예시 업데이트
docs(setup): uv 설치 지침 추가
docs: 기여 가이드의 오타 수정
```

### 보조 타입

#### `style` - 코드 스타일
- 포맷팅 변경
- 세미콜론 누락
- 공백 변경
- 코드 스타일 개선

```bash
style(backend): black으로 코드 포맷팅
style(frontend): ktlint 경고 수정
style: 후행 공백 제거
```

#### `refactor` - 코드 리팩토링
- 기능 변경 없는 코드 재구성
- 성능 개선
- 코드 조직화

```bash
refactor(auth): 검증 로직을 별도 모듈로 추출
refactor(ai): 모델 로딩 성능 최적화
refactor: 유틸리티 함수 재조직
```

#### `test` - 테스팅
- 테스트 추가
- 기존 테스트 업데이트
- 테스트 설정 변경

```bash
test(auth): JWT 검증을 위한 단위 테스트 추가
test(ai): 모델 파이프라인을 위한 통합 테스트 추가
test: 사용자 서비스 커버리지 증가
```

#### `chore` - 유지보수
- 의존성 업데이트
- 빌드 설정
- CI/CD 변경
- 도구 설정

```bash
chore(deps): fastapi를 0.104.0으로 업데이트
chore(ci): 보안 스캔 워크플로우 추가
chore: pre-commit hooks 설정 업데이트
```

#### `perf` - 성능
- 성능 개선
- 최적화 변경
- 리소스 사용량 개선

```bash
perf(api): 데이터베이스 쿼리 성능 최적화
perf(ai): 모델 추론 시간 단축
perf: 빈번한 요청에 대한 캐싱 구현
```

#### `ci` - 지속적 통합
- CI/CD 파이프라인 변경
- GitHub Actions 업데이트
- 빌드 스크립트 수정

```bash
ci: 자동화된 테스팅 워크플로우 추가
ci(deploy): 프로덕션 배포 스크립트 업데이트
ci: docker 빌드 캐싱 이슈 수정
```

#### `build` - 빌드 시스템
- 빌드 도구 변경
- 의존성 관리
- 패키지 설정

```bash
build: pyproject.toml 설정 업데이트
build(docker): 컨테이너 이미지 크기 최적화
build: uv lock 파일 생성 추가
```

#### `revert` - 되돌리기
- 이전 커밋 되돌리기
- 변경사항 롤백

```bash
revert: "feat(auth): OAuth 통합 추가" 되돌리기
revert: 데이터베이스 마이그레이션 변경사항 롤백
```

## 🎯 스코프 가이드라인

스코프는 변경 위치에 대한 추가 컨텍스트를 제공합니다:

### 백엔드 스코프
- `api` - API 엔드포인트 및 라우트
- `auth` - 인증 및 권한 부여
- `db` - 데이터베이스 모델 및 마이그레이션
- `service` - 비즈니스 로직 서비스
- `config` - 설정 파일
- `middleware` - 미들웨어 컴포넌트

### 프론트엔드 스코프
- `ui` - 사용자 인터페이스 컴포넌트
- `navigation` - 앱 네비게이션
- `auth` - 인증 플로우
- `api` - API 통합
- `utils` - 유틸리티 함수
- `theme` - 스타일링 및 테마

### AI 모델 스코프
- `model` - 모델 아키텍처 및 훈련
- `data` - 데이터 처리 및 준비
- `inference` - 모델 추론 및 서빙
- `training` - 훈련 스크립트 및 파이프라인
- `evaluation` - 모델 평가 및 메트릭

### 일반 스코프
- `docs` - 문서
- `tests` - 테스팅
- `ci` - 지속적 통합
- `deps` - 의존성
- `config` - 설정

## ✍️ 작성 가이드라인

### 설명 규칙
1. **명령형 사용**: "추가" (과거형 "추가했음"이나 현재형 "추가함" 아님)
2. **소문자로 시작**: "사용자 로그인 수정" (대문자 "사용자 로그인 수정" 아님)
3. **끝에 마침표 없음**: "새 기능 추가" ("새 기능 추가." 아님)
4. **구체적으로**: "로그인 버그 수정" → "비밀번호 검증 오류 수정"
5. **짧게 유지**: 헤더는 최대 50자

### 본문 가이드라인
1. **왜를 설명**: 왜 이 변경이 필요했는가?
2. **컨텍스트 제공**: 어떤 문제를 해결하는가?
3. **부작용 언급**: 중대한 변경사항이나 영향
4. **이슈 참조**: 관련 이슈나 티켓 링크

### 푸터 가이드라인
1. **중대한 변경사항**: "BREAKING CHANGE:"로 시작
2. **이슈 참조**: "Closes #123", "Fixes #456", "Refs #789"
3. **공동 작성자**: "Co-authored-by: 이름 <email@example.com>"

## 📚 예시

### 간단한 커밋
```bash
feat(auth): 비밀번호 재설정 기능 추가
```

### 본문이 있는 커밋
```bash
fix(api): 사용자 생성의 경쟁 상태 해결

사용자 생성 엔드포인트에서 동시에 여러 요청이 들어올 때
경쟁 상태가 발생했습니다. 이 수정은 중복 사용자 생성을
방지하기 위한 적절한 잠금 메커니즘을 추가합니다.

Fixes #234
```

### 중대한 변경사항
```bash
feat(api): 인증 엔드포인트 재설계

BREAKING CHANGE: 인증 API가 재설계되었습니다.
/login 엔드포인트가 이제 다른 응답 형식을 반환합니다.
클라이언트 코드를 그에 맞게 업데이트하세요.

- 응답 형식을 {token}에서 {access_token, refresh_token}으로 변경
- 토큰 만료 정보 추가
- 더 이상 사용되지 않는 /auth/validate 엔드포인트 제거

Closes #123
```

### 여러 변경사항
```bash
feat(ai): 텍스트 분류 모델 구현

- BERT 기반 텍스트 분류기 추가
- 조기 중단을 포함한 훈련 파이프라인 구현
- 모델 평가 메트릭 추가
- 추론 API 엔드포인트 생성

모델은 테스트 데이터셋에서 94% 정확도를 달성하며
100ms 미만의 응답 시간으로 실시간 분류를 지원합니다.

Closes #456, #789
```

### 되돌리기 커밋
```bash
revert: "feat(auth): OAuth 통합 추가" 되돌리기

OAuth 구현에서 식별된 보안 문제로 인해
커밋 1234567890abcdef를 되돌립니다. 적절한 보안 조치와 함께
다시 구현될 예정입니다.

Refs #999
```

## 🔧 도구 및 자동화

### Git 커밋 템플릿
커밋 메시지 템플릿이 자동으로 설정됩니다:

```bash
# .gitmessage 템플릿이 자동으로 설정됨
git config commit.template .gitmessage
```

### Pre-commit Hooks
Pre-commit hooks가 커밋 메시지를 검증합니다:

```yaml
# .pre-commit-config.yaml에 커밋 메시지 검증 포함
- repo: https://github.com/compilerla/conventional-pre-commit
  rev: v2.4.0
  hooks:
    - id: conventional-pre-commit
      stages: [commit-msg]
```

### 커밋 메시지 검증
```bash
# 커밋 메시지 형식 검증
npx commitlint --from HEAD~1 --to HEAD --verbose

# 검증과 함께 대화형 커밋
npx git-cz
```

### IDE 통합
- **VS Code**: "Conventional Commits" 확장 설치
- **IntelliJ/Android Studio**: "Git Commit Template" 플러그인 설치
- **Vim**: 커밋 메시지 템플릿 사용

## ❌ 일반적인 실수

### ❌ 나쁜 예시
```bash
# 너무 모호함
fix: 버그 수정

# 잘못된 시제
feat: 새 기능을 추가했음

# 너무 김
feat(auth): JWT 토큰과 갱신 메커니즘을 포함한 포괄적인 사용자 인증 시스템 구현

# 타입 누락
사용자 서비스 업데이트

# 잘못된 대소문자
Fix: 사용자 로그인 이슈

# 끝에 마침표
feat(ui): 다크 모드 추가.

# 명령형이 아님
feat(api): 새 엔드포인트를 추가하는 중
```

### ✅ 좋은 예시
```bash
# 명확하고 구체적
fix(auth): 비밀번호 검증 오류 해결

# 적절한 명령형
feat(ui): 다크 모드 토글 추가

# 적절한 길이
feat(auth): JWT 갱신 메커니즘 구현

# 적절한 형식
refactor(api): 사용자 검증 로직 추출

# 올바른 대소문자
fix(db): 연결 타임아웃 이슈 해결

# 마침표 없음
feat(ai): 감정 분석 모델 추가

# 명령형
feat(api): 사용자 프로필 엔드포인트 추가
```

## 🔍 커밋 메시지 체크리스트

커밋하기 전에 메시지가 다음을 만족하는지 확인하세요:

- [ ] 관례적 커밋 형식을 따름
- [ ] 적절한 타입과 스코프 사용
- [ ] 명확하고 설명적인 제목 줄
- [ ] 명령형 사용 ("추가" not "추가했음")
- [ ] 소문자로 시작 (고유명사 제외)
- [ ] 제목 끝에 마침표 없음
- [ ] 제목 줄이 50자 미만
- [ ] 복잡한 변경사항인 경우 본문 포함
- [ ] 해당하는 경우 관련 이슈 참조
- [ ] 해당하는 경우 중대한 변경사항 언급

## 🚀 고급 사용법

### 시맨틱 버전 통합
커밋 타입이 자동으로 버전 범프를 결정합니다:
- `feat`: 마이너 버전 범프 (1.1.0 → 1.2.0)
- `fix`: 패치 버전 범프 (1.1.0 → 1.1.1)
- `BREAKING CHANGE`: 메이저 버전 범프 (1.1.0 → 2.0.0)

### 자동 변경로그
관례적 커밋은 자동 변경로그 생성을 가능하게 합니다:
```bash
# 커밋에서 변경로그 생성
npx conventional-changelog -p angular -i CHANGELOG.md -s
```

### 릴리스 자동화
커밋이 자동 릴리스를 트리거합니다:
```bash
# 커밋 메시지 기반 시맨틱 릴리스
npx semantic-release
```
